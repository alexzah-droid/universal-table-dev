# Покрытие тестов — Universal Table

## Запуск

```bash
npm test                  # все тесты (desktop + mobile)
npm run test:desktop      # только десктоп
npm run test:mobile       # только мобайл
npm run test:headed       # с открытым браузером
```

**79 тестов:** 65 desktop + 14 mobile

---

## Архитектура тестов

Фикстуры генерируются автоматически в `tests/fixtures/` перед запуском (`beforeAll`). Три предметные области:
- **Контакты** (10 строк): name/company/role[select]/date/level[select], expand: null
- **Задачи** (8 строк): title/priority[select]/due/status[select]/points[number], expand: notes (Markdown)
- **Пустая таблица** (0 строк): name/category[select, allowNew]/amount[number], expand: notes

## 1. Setup wizard `@desktop` — 13 тестов

- `#setup-page` виден, `#app` скрыт при CONFIG=null
- Форма содержит поля title, filename, кнопку «Создать»
- По умолчанию 1 карточка столбца
- Добавление до 5 столбцов; кнопка блокируется при 5
- Удаление карточки столбца
- Тип «Список» показывает textarea для вариантов
- Тип «Дата» и «Число» блокируют чекбокс «Фильтр»
- Валидация: пустой title / пустое название столбца → `.error`
- Toggle expand-поля включает/отключает поле названия
- Заполненная форма скачивает файл с корректным CONFIG

## 2. Данные: контакты `@desktop` — 28 тестов

**Read (7):**
- app-page виден, setup скрыт
- Заголовок таблицы из CONFIG.title
- Счётчик (`#subtitle-count`) = 10
- 10 строк в `#list`
- Заголовки столбцов соответствуют CONFIG.columns
- Первый столбец имеет стиль `primary` (font-weight: 600)
- Дата отображается в формате DD.MM.YYYY

**Filter (8):**
- Кнопка «Сбросить» скрыта при отсутствии фильтров
- Фильтр по Компании (text-type) сужает список
- Фильтр по Роли (select-type) сужает список
- Пересечение двух фильтров
- Нулевой результат → empty-state
- Кнопка «Сбросить» видна при активном фильтре
- Кнопка «Сбросить» восстанавливает полный список
- После добавления новая компания появляется в фильтре

**Sort (6):**
- «Имя: А–Я» — первая запись имеет наименьшее имя
- «Имя: Я–А» — первая запись имеет наибольшее имя
- «Дата: ↑ старые» — возрастающий порядок
- «Дата: ↓ новые» (default) — убывающий порядок
- «Компания: А–Я» — весь список отсортирован
- «По номеру ↑» — первая запись `data-id=1`

**CRUD Create (4):**
- Кнопка «+ Добавить» открывает модальный диалог
- Валидация: пустой primary-поле не закрывает модал
- Успешное добавление: запись появляется, счётчик растёт
- После добавления появляется индикатор «Есть изменения»

**CRUD Update (3):**
- Кнопка карандаша открывает форму с предзаполненными данными
- Редактирование имени обновляет список
- Клик «Редактировать» не раскрывает аккордеон

## 5. Тема и брейкпоинт `@desktop` — 8 тестов

**Theme (4):**
- Клик на иконку темы → `data-theme="dark"`
- Повторный клик → `"light"`
- Иконки sun/moon переключаются
- Тема сохраняется в `localStorage` (`ut-theme`) и восстанавливается

**Breakpoint (4):**
- 769px → desktop layout (`#filter-bar` виден)
- 768px → mobile layout (`#topbar` виден)
- Resize desktop → mobile без перезагрузки
- Resize mobile → desktop без перезагрузки

## Что намеренно не тестируется

- **Сохранение файла (File System Access API / download):** headless-браузер не предоставляет реальной файловой системы. `buildHtml()` тестируется косвенно — после каждой мутации `rows` обновляется корректно.
- **`beforeunload` предупреждение:** браузеры блокируют программную навигацию при наличии listener — тест был бы ненадёжен.
- **Таймер 3 секунды на сброс delete-confirm:** потребует `waitForTimeout(3000+)` — замедляет сьют без ценности.
- **Содержимое XLSX-файла:** `page.waitForEvent('download')` проверяет факт скачивания; парсить ZIP в headless нецелесообразно.
